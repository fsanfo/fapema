Contrato do design system

nao pode regredir entre telas
Grupo Contrato Risco se quebrar
Tokens brand-primary, brand-secondary, paper/panel, estados semanticos Perda de consistencia visual
Tipografia League Spartan para titulos, Poppins para corpo e metadados Queda de hierarquia editorial
Shell Sidebar fixa + coluna principal + breadcrumb + meta-pill Quebra de navegacao entre paineis
Componentes Cards, tabelas, badges, chips e estados active/hover/focus Inconsistencia de interacao

Acessibilidade e responsividade

criterio de homologacao obrigatorio
  • A11y Manter lang pt-BR, skip-link e foco visivel de alto contraste.
  • Teclado Toda navegacao e decisoes precisam ser operaveis sem mouse.
  • Semantica Estados nao podem depender apenas de cor.
  • Mobile Grids devem colapsar sem perda de leitura de KPI e filtros.
  • Tabela Permitir rolagem horizontal segura em densidade alta.

Definition of done (UX para DEV)

sinal de pronto para avancar para homologacao
Navegacao e shell consistentes

Todas as telas compartilham sidebar, breadcrumb, estados active/hover/focus e hierarquia de layout.

Obrigatorio
Paridade visual com V2

Tokens, tipografia, semaforizacao e contexto visual preservados sem simplificacoes nao aprovadas.

Obrigatorio
Fallback de dados e estados vazios

Implementar estados parcial/falho/pendente em conciliacao e homologacao para operacao segura.

Alta prioridade
Rastreabilidade insuficiente

Bloqueia aceite se nao houver conexao clara entre indicador, regra e origem de dados.
